# [PromptSkiller] PRD（Web：MVP -> MVP+1）

## 1. 产品问题

提示词能力（Prompting）正在成为基础能力，但多数用户缺少系统训练，常见问题是：

- 上下文不完整，导致 AI 猜测过多。
- 约束与验收标准不明确，输出不可控。
- 不会基于反馈迭代提示词，停留在“一次提问”。

PromptSkiller 的目标是把“提示词训练”做成可持续的日常练习，而不是一次性问答工具。

## 2. 产品定位

PromptSkiller 是一个「提示词训练器」：

- 让用户自己写提示词。
- AI 教练只负责评估和引导优化，不直接替代用户思考。
- 强调“多轮迭代”与“可验证表达”。

## 3. 目标用户

- 核心用户：开发者、计算机专业学生、重度 AI 工具使用者。
- 次级用户：希望提升结构化表达能力的产品/运营/设计同学。

## 4. 当前范围（2026-02-28）

### 已上线（MVP）

- 账号体系：登录/注册（Supabase Auth）。
- 训练模块：题库、今日训练、提示词提交、教练反馈、历史记录。
- 比赛模块：周赛列表/详情、作品提交、投票、公开分享页。
- 个人主页：训练统计 + 我的周赛提交与获赞。
- 管理后台（最小版）：新增题目、编辑/发布/下线、每日排期、提交统计。
- 模型配置：支持 Mock / OpenAI / 百炼 / 自定义兼容服务。

### 已上线（MVP+1）

- 四题型并行：`prompt_case` / `code_case_multi` / `build_sim_case` / `template_case`。
- 复杂题附件：`题面/文件/日志` 三类附件上下文展示。
- 教学样板题：`template_case` 只读看板（固定 2-3 轮提示词迭代展示，不支持提交）。
- 训练反馈模式：`过程引导` / `终局评分`。
- 反馈模式默认值可配置；终局模式支持中途查看“简版评分”（仅分数）。
- 会话轮次模型：每轮评估持久化并支持终局统一复盘。
- 模块学习路径：`drill_modules` + `drill_module_items` 最小版上线。
- 题库新增双视图：题库视图 + 模块视图，支持模块过滤和云端进度展示（登录态）。

### 暂不包含

- 支付订阅与商业化计费。
- 完整后台 CMS（审核流、内容协作工作流、批量导入导出）。
- 强防作弊体系（目前采用荣誉规则 + 透明 prompt log）。
- 真实线上文件系统/沙箱执行（`build_sim_case` 首版采用虚拟工作区快照）。

## 5. 关键产品规则（截至 2026-02-28）

- 训练题展示：全站递增题号（`PS-001`）。
- 今日训练：每天推荐 3 题（优先管理员题单，缺口自动补齐）。
- 同一题支持多次提交和历史回看。
- 会话模式：
- `guided`：每轮即时可见评分。
- `final_only`：每轮先入库隐藏，完成会话后统一显示评分和复盘。
- 周赛规则：每位用户每周每题 1 次提交。
- 投票规则：一人对同一作品最多 1 票，支持取消投票，不可给自己投票。

## 6. AI 教练输出规范

结构化评分字段：

- `score_total`（0-120）
- `scores.context`
- `scores.constraints`
- `scores.output_format`
- `scores.acceptance_criteria`
- `scores.tests_and_edge_cases`
- `scores.process_control`
- `missing_items[]`
- `ambiguities[]`
- `suggested_questions_to_answer[]`
- `rewrite_example`（可选，默认折叠展示）

`build_sim_case` 额外结构化产物字段：

- `summary`
- `changed_files[]`
- `patch_preview`
- `risk_notes[]`

## 7. 成功指标（当前阶段）

- D1 / D7 留存。
- 训练提交率（打开训练页 -> 完成至少 1 次提交）。
- 单题平均迭代次数（同题多次提交）。
- 模块启动率（进入模块视图 -> 至少打开 1 道模块题）。
- 终局评分完成率（`final_only` 会话中点击“完成会话”比例）。
- 周赛参与率与互动率。

## 8. 风险与控制

- 成本风险：真实模型调用成本不可控。
- 控制策略：默认 Mock、真实模式需显式配置 key、后续加入限流与预算策略。
- 质量风险：反馈模板化、学习价值下降。
- 控制策略：持续优化 rubric + 题型覆盖 + 终局复盘质量。
- 安全风险：用户误贴隐私/密钥。
- 控制策略：UI 提示 + 服务端校验长度与基础防护。
- 复杂度风险：模拟构建题若过早引入真实沙箱会显著升高工程成本。
- 控制策略：先用虚拟工作区验证价值，再决定是否进阶。
