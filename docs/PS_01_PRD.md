# [PromptSkiller] PRD（Web MVP）

## 1. 产品问题

提示词能力（Prompting）正在成为基础能力，但多数用户缺少系统训练，常见问题是：

- 上下文不完整，导致 AI 猜测过多。
- 约束与验收标准不明确，输出不可控。
- 不会基于反馈迭代提示词，停留在“一次提问”。

PromptSkiller 的目标是把“提示词训练”做成可持续的日常练习，而不是一次性问答工具。

## 2. 产品定位

PromptSkiller 是一个「提示词训练器」：

- 让用户自己写提示词。
- AI 教练只负责评估和引导优化，不直接替代用户思考。
- 强调“多轮迭代”与“可验证表达”。

## 3. 目标用户

- 核心用户：开发者、计算机专业学生、重度 AI 工具使用者。
- 次级用户：希望提升结构化表达能力的产品/运营/设计同学。

## 4. MVP 范围（冻结）

### 包含

- 账号体系：登录/注册（Supabase Auth）。
- 训练模块：题库列表与搜索。
- 训练模块：今日训练（每日推荐 3 题）。
- 训练模块：训练题详情与固定链接。
- 训练模块：提示词提交、教练反馈、历史记录。
- 比赛模块：周赛列表与详情。
- 比赛模块：作品提交（URL/Repo/截图链接等）+ Prompt Log。
- 比赛模块：点赞投票与公开分享页。
- 个人主页：训练次数、平均分、连续天数、我的周赛提交和获赞。
- 模型配置：默认 Mock。
- 模型配置：支持 OpenAI / 百炼 / 自定义兼容服务。
- 模型配置：API Key 联通性测试与保存。

### 不包含

- 支付订阅与商业化计费。
- 后台管理系统（当前依赖 SQL migration + seed）。
- 强防作弊体系（目前采用荣誉规则 + 透明 prompt log）。
- 原生 App 打包。

## 5. 关键产品规则（截至 2026-02-28）

- 训练题展示：全站递增题号（`PS-001`）。
- 今日训练：每天推荐 3 题，用户可任选一题开始。
- 同一题可多次提交提示词版本，支持历史回看。
- 周赛规则：每位用户每周每题 1 次提交。
- 投票规则：一人对同一作品最多 1 票，支持取消投票，不可给自己投票。

## 6. AI 教练输出规范（MVP）

结构化输出字段：

- `score_total`（0-100）
- `scores.context`
- `scores.constraints`
- `scores.output_format`
- `scores.acceptance_criteria`
- `scores.tests_and_edge_cases`
- `missing_items[]`
- `ambiguities[]`
- `suggested_questions_to_answer[]`
- `rewrite_example`（可选，默认折叠展示）

## 7. 成功指标（MVP）

- D1/D7 留存。
- 训练提交率（打开训练页 -> 完成至少 1 次提交）。
- 单题平均迭代次数（同题多次提交）。
- 周赛参与率（有账号用户中参与提交比例）。
- 周赛互动率（提交作品的平均获赞与投票人数）。

## 8. 风险与控制

- 成本风险：真实模型调用成本不可控。
- 控制策略：默认 Mock、真实模式需显式配置 key、后续增加限流。
- 质量风险：反馈模板化、用户学习价值下降。
- 控制策略：持续优化 rubric 与引导文案，关注“能否推动二次改写”。
- 安全风险：用户误贴隐私/密钥。
- 控制策略：UI 提示 + 服务端校验长度与基础防护。
