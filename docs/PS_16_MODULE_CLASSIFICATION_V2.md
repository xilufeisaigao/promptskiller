# [PromptSkiller] 模块分类页 V2 设计与落地（2026-03-01）

## 1. 目标

把题库从“可搜索列表”升级为“可导航能力地图”，让用户先进入专题，再开始训练。

- 支持按题型分类与按标签分类两种入口。
- 支持类似成熟刷题平台的分面筛选（题型 / 难度 / 练习状态 / 发布时间）。
- 保留模块学习路径，并增加按模块等级过滤。

## 2. 参考成熟刷题平台的抽象做法

对标能力不是照搬 UI，而是照搬信息架构：

- Problemset 的核心是“多维分面 + 快速命中目标题”。
- Topic/Tag 的核心是“专题入口 + 题量规模感”。
- Study Plan 的核心是“分级路径 + 连续训练进度”。

基于 PromptSkiller 的当前数据条件（题型、标签、难度、发布时间、用户练习进度），V2 做法为：

1. 分面筛选层：练习状态、难度、题型、新题开关、关键词、排序。
2. 专题发现层：
   - 按题型专题：普通题 / 多文件题 / 模拟构建题 / 教学样板题。
   - 按标签专题：展示标签卡片及题量、近期新增量。
3. 模块路径层：继续提供模块卡片与进度条，新增 Starter/Intermediate/Advanced 等级筛选。

## 3. 本轮实现范围（MVP+1）

### 3.1 已实现

- `/drills` 新增“模块化分类导航”区块：
  - 左侧分面控制：练习状态、难度分层、题型分层。
  - 右侧专题区：按题型专题 / 按标签专题可切换。
- 题库过滤逻辑升级：
  - 新增题型多选过滤（AND with existing filters）。
  - 新增难度多选过滤。
  - 新增练习状态过滤（全部 / 已练 / 未练）。
- 模块视图增强：
  - 新增模块等级过滤（all/starter/intermediate/advanced）。
- 结果区增强：
  - 标签筛选按钮附带题量。
  - 显示当前已启用的筛选条件数量。

### 3.2 本轮未做（留给后续）

- URL 参数与筛选状态双向同步（分享同一筛选视图）。
- “通过率/热门度/提交人数”排序（当前数据模型未沉淀这些指标）。
- 标签体系的层级分组页（目前仍为扁平 tag）。

## 4. 验收清单

1. 用户可在 2-4 次点击内定位：
   - 某题型 + 某难度 + 未练题。
2. 用户可从标签专题快速进入对应题集，并继续叠加筛选。
3. 模块视图可按等级过滤并保留原有学习路径体验。
4. 页面在移动端与桌面端都保持清晰信息层级。

## 5. 后续建议（V2.1）

1. 增加筛选状态写回 URL（`/drills?types=...&tags=...`），支持分享与回访。
2. 后台新增标签词表管理（防止同义标签碎片化）。
3. 引入“完成率/平均分”排序，进一步接近成熟刷题站的运营能力。
