# [PromptSkiller] 周赛机制（竞赛规范）

## 目标

把提示词训练从“个人练习”扩展到“可展示、可复盘、可互动”的社区玩法，但不做重赛制的编程平台。

## 当前已实现规则（2026-02-28）

- 频率：每周 1 个挑战题。
- 参与资格：登录用户可提交。
- 提交限制：每个挑战每个用户最多 1 份提交（数据库唯一约束保证）。
- 提交字段：`artifact_url`（作品链接）。
- 提交字段：`artifact_type`（`url` / `image` / `repo` 等）。
- 提交字段：`prompt_log_md`（关键提示词）。
- 提交字段：`notes`（可选说明）。
- 投票规则：支持点赞/取消点赞切换。
- 投票规则：每人对同一作品最多 1 票。
- 投票规则：不能给自己的作品投票。
- 展示规则：周赛详情页展示所有提交与票数。
- 展示规则：每份提交有独立公开分享页。

## “不手写代码”规则

该规则当前按荣誉机制执行，不做强制检测。

执行方式：

- 强调 Prompt Log 透明度（让他人能看见过程）。
- 鼓励提交复现说明和迭代记录。
- 投票时重点参考“提示词结构化程度”，不只看最终结果。

## 当前不做（后续迭代项）

- 评论系统与讨论串。
- 多奖项评选（例如最佳提示词、最佳复盘）。
- 复杂赛制（淘汰、分组、权重评分）。
- 防刷票与反作弊策略（当前仅基础唯一约束 + RLS）。

## 后续可扩展方向

- 增加“最佳提示词清晰度/最佳复盘”奖项标签。
- 增加赛后复盘页（高票作品 + 关键提示词拆解）。
- 增加提交版本历史（v1/v2/v3）以强化训练价值。
