# [PromptSkiller] 周赛机制（竞赛规范）

## 目标

让提示词练习具备社交性与结果导向，但不变成完整的 LeetCode 复制品。

## MVP 规则

- 每周 1 个挑战题
- 每人 1 次提交
- 提交必须包含：
  - 作品（URL 或截图）
  - prompt log（关键提示词）
  - 简短说明（可选）
- 投票：
  - 只做点赞
  - 每人对同一作品最多点赞一次

## “不允许自己写一行代码”

强制执行很难。MVP 处理方式：

- 先当作荣誉规则（honor system）。
- 强调透明度：
  - 必填 prompt log
  - 提供复现步骤
- 需要时再扩展：
  - 增加奖项/类别：“最佳提示词”而不是“最佳代码”

## 题目示例（先从编程领域开始）

- “做一个响应式 UI 卡片组件，包含表单 + 校验”
- “为某个函数写测试（单元测试 + 边界情况）”
- “在保证行为不变的前提下重构一段混乱代码”
- “设计一个 API 规格并生成 client”

## 待定问题

- 默认是否公开所有提交？
- 可以由用户选择
- 是否支持评论？
- 支持，但现在不支持多层评论（即对评论的回复的回复）
- 是否提供多个奖项（最佳结果 / 最佳提示词 / 最佳说明）？
- 可以，你做成可拓展的
